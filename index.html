GIT

Git is a distributed version control system(DVCS), meaning it keeps track of changes in your files and allows you to revert to previous versions if needed

Git Workflow- Local Repository

  1. You make changes to your files in the working directory
  2. You then stage the specific changes you want to keep for the next version
  3. Finally you commit the staged changes, creating a new snapshot in your local repository

Branching: Git allows you to create branches of your project, essentially creating temporary copies to work on specific features or bug fixes without affecting the main project. Once you are happy with your changes in a branch you can merge them back into the main branch
